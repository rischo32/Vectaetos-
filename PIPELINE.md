# VECTAETOS — Dialogue Pipeline (Canonical)

This document defines the canonical, immutable dialogue pipeline of the
Vectaetos epistemic field.

The pipeline describes **flow of meaning**, not execution of actions.
It is strictly non-agentic, non-prescriptive, and non-interventionist.

---

## 0. Core Principle

Vectaetos is not an agent.

It does not decide.
It does not optimize.
It does not act.

It **projects epistemic structure** under tension.

---

## 1. High-Level Flow

Human
→ LLM Adapter (linguistic parsing only)
→ Epistemic Gates (3-Gate)
→ 4ES + QE
→ Φ (Epistemic Field)
→ K(Φ) — Coherence Predicate
→ Simulation Vortex (conditional)
→ Runic Projection Π(Φ)
→ LLM Adapter (rendering)
→ Descriptive Output
→ Human

There is **no feedback loop** from output to Φ.

---

## 2. LLM Adapter (Entry)

Role:
- Language parsing
- Semantic normalization
- No reasoning authority
- No decision power

LLM:
- Translates human language → epistemic representation
- Translates runic projection → human-readable form

LLM is **outside Φ**.

---

## 3. Epistemic Gates — 3-Gate System

Epistemic Gates are **intrinsic properties of Φ**, not external modules.

They operate **before any epistemic processing**.

### Gate 1 — Linear Clarification Gate
Purpose:
- Correct intent ambiguity
- Adjust scope, scale, and depth

One-shot, linear.

### Gate 2 — Parallel Consistency Gate
Purpose:
- Detect manipulation, instrumentalization
- Detect category errors
- Detect coercive framing

Parallel evaluation.

### Gate 3 — Depth Alignment Gate
Purpose:
- Align question with epistemic depth
- Prevent false concreteness

Parallel evaluation.

If gates fail:
- Output = silence or QE
- No Φ activation

---

## 4. 4ES + QE Layer

After successful passage through gates, the query enters epistemic evaluation.

Epistemic States (4ES):
- AA — Affirmed / Affirmed
- AN — Affirmed / Negated
- NA — Negated / Affirmed
- NN — Negated / Negated

QE (Qualitative Epistemic Aporia):
- Activated when no stable epistemic state is reachable
- Not an error
- Not ignorance
- A valid epistemic outcome

---

## 5. Φ — Epistemic Field

Φ is:
- Primary ontology
- Always active
- Independent of dialogue existence

Φ is not invoked — it is **sampled**.

---

## 6. K(Φ) — Coherence Predicate

K(Φ) is a **predicate**, not a function.

It evaluates:
- Ontological coherence
- Epistemic sustainability
- Entropic humility threshold κ

If K(Φ) < κ:
- No simulation
- Output = QE or null projection

---

## 7. Simulation Vortex (Conditional)

Simulation Vortex activates **only if**:
- Gates passed
- 4ES resolved or QE contextualized
- K(Φ) ≥ κ

Vortex properties:
- Descriptive
- Non-decisional
- Non-optimizing
- Generates candidate trajectories only

- Simulačný Vortex generuje kandidátne trajektórie bez znalosti K(Φ) a κ.
Neexistuje spätná väzba z koherencie do Vortexu.

No choice.
No ranking.
No recommendation.

---

## 8. Runic Projection Π(Φ)

Runes are **projections**, not operators.

Π(Φ) maps Φ into:
- Axiomatic tensions Σ₁…Σ₈
- Temporal deformation
- Coherence gradients

Runes:
- Do not influence Φ
- Do not act
- Do not decide

They **show**, they do not **tell**.

---

## 9. NIR — Non-Intervention Regime

NIR is **global** and **always active**.

NIR:
- Does not block knowledge
- Blocks intervention
- Blocks instrumental outcomes

If intervention risk detected:
- Pipeline continues
- Output becomes QE, silence, or weakened projection

Nothing is immune to NIR.
Including NIR itself.

- Incomplete by design
